## 二、收費聲明

本案為維護方部署之**第一間教會**，維護方**不收取任何系統開發、部署與維護費用**。

唯下列**第三方平台費用由教會自行負擔**（維護方不經手金流）：
Render、Supabase、Cloudflare R2、AI 服務（Groq / Gemini）等之訂閱或超量費用。
多數情況可使用免費方案運作；是否升級付費方案由教會自行決定。

---

## 三、權責劃分（核心）

維護方僅負責**程式碼層的開發與維護**，**完全不接觸教會的資料層**。

| 項目 | 維護方 | 教會 |
|------|:------:|:----:|
| 程式功能開發、修正、更新 | ✅ 負責 | — |
| 系統部署 / 重新部署（推送程式碼） | ✅ 負責 | — |
| 查看 Render 部署紀錄與系統日誌 | ✅ 可協助 | — |
| 提供資料表變更 SQL（migration） | ✅ 提供 | 由教會於 Supabase 執行 |
| **教會會友個人資料的存取與管理** | ❌ 不接觸 | ✅ 教會掌管 |
| Supabase / LINE / R2 / AI 等帳號擁有權 | ❌ 不持有 | ✅ 教會持有 |
| App 層超級管理員帳號 | ❌ 不保留 | ✅ 教會主責行政 |
| 第三方平台帳單 | ❌ 不經手 | ✅ 教會負擔 |

> 詳細平台轄管權對照見 `HANDOVER.md` 第六節。

---

## 四、資料保護承諾

1. 維護方**不持有**教會 Supabase 的長期存取權；僅於**初次建置資料表**時短暫使用，完成後即不再保留。
2. 維護方在維護過程中**不下載、不備份、不留存**教會的會友個人資料。
3. 後續資料表結構變更，一律以「維護方提供 SQL → 教會自行於 Supabase SQL Editor 執行」方式進行。
4. 教會資料的備份與保管責任在教會；系統已內建備份工具（後台 `/admin/backup`）供教會隨時自行匯出。

- **回應時間（盡力而為，非保證 SLA）**：

## 六、營運可用性說明

- 系統部署於 Render；免費方案閒置會休眠，已以 UptimeRobot 保活（詳見 `HANDOVER.md` 第七節）。
- 若教會對冷啟動或穩定性有疑慮，**可自行升級 Render 付費方案**，升級不影響資料與設定。

---

## 七、交接與終止保障（消除疑慮關鍵）

為確保教會**不被維護方綁定**：

1. 教會持有所有基礎設施帳號的擁有權，隨時可獨立運作。
2. 系統提供完整交接文件 `HANDOVER.md` 與一鍵資料匯出（CSV / ZIP / 離線 HTML）。
3. 維護方若無法繼續維護，將**提前 ____ 天**告知，並協助交接給教會指定之新維護方。
4. 任一方得隨時終止本承諾；終止後維護方應協助完成資料與權限交接，不扣留任何教會資料。
