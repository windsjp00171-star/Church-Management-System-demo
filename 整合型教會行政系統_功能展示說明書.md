# 整合型教會行政系統
## 功能展示說明書
*2026年6月版 · 展示用途*

---

## 一、這套系統是怎麼做出來的？

### 開發方式：Vibe Coding（AI 協作開發）

這套系統採用當前最新的 **AI 協作開發模式（Vibe Coding）**，由牧師與工程師共同描述需求，AI 輔助生成與優化程式碼，再由人工審核部署。

這代表什麼？

- **開發速度極快**：原本需要三到六個月的系統，在數週內可推出可用版本
- **需求驅動，而非技術驅動**：每一個功能都從「教會實際遇到的問題」出發，不是為了炫技
- **持續迭代**：教會提出新需求，系統可快速調整，不需漫長的開發週期

這不是「買現成軟體」，也不是傳統「外包給工程師」——而是教會與 AI 協作，打造出真正符合自己需要的工具。

## 二、登入層：零門檻進入

### 登入方式：LINE 帳號一鍵登入

台灣教會幾乎人人有 LINE。本系統以 **LINE 帳號作為唯一登入憑證**：

| 項目 | 說明 |
|------|------|
| 登入方式 | 點「LINE 登入」，授權一次即完成 |
| 免記帳號密碼 | 不需另外申請帳號 |
| 免下載 App | 瀏覽器直接使用，加到主畫面即變 App |
| 身份自動辨識 | 登入後系統自動判斷角色（一般會友／同工／牧師／超管） |

### 跨平台優勢

| 裝置 | 使用方式 |
|------|----------|
| 手機（iOS / Android） | 瀏覽器開啟，加到主畫面，體驗等同原生 App |
| 平板 | 自動適應版面 |
| 電腦 | 完整後台功能 |

**不需要** 上架 App Store、不需要使用者更新版本、不需要考慮 iOS / Android 相容性。教會換新功能，會友下次打開就直接用到新版。

### 身份層級

```
超管（牧師）→ 管理員（同工）→ 一般會友 → 訪客（無帳號）
```

每個層級看到的功能不同，訪客連結可讓未登入的訪客填表（報名、週報出席等）。

### 模塊 1：活動報名系統

**解決的問題：** 用 LINE 群組收報名，遺漏人名、無法統計、付款狀況難追蹤。

**功能說明：**
- 管理員建立活動，自訂報名表單欄位
- 會友點連結填表，系統即時記錄
- 後台一覽名單，可匯出 Excel
- 報名截止前自動提醒尚未繳費者
- 支援活動封面圖上傳

**市場比較：**

| 工具 | 限制 |
|------|------|
| Google 表單 | 要另開試算表整理、無自動提醒、無身份整合 |
| Accupass / Kktix | 面向公開商業活動，有手續費，非教會設計 |
| LINE 官方帳號報名 | 需另建 Webhook，開發門檻高 |
| **本系統** | **與會友帳號整合，免重複填資料，自動對接繳費提醒** |

### 模塊 1.5：線上金流收款 ⭐ 新功能

**解決的問題：** 活動收費靠現金或轉帳，財務難追蹤、對帳耗時、繳費提醒靠人工。

**功能說明：**
- 每個活動可個別開啟或關閉線上繳費，不是全教會統一開關
- 支援三種金流：**綠界 ECPay**（信用卡 / ATM / 超商）、**LINE Pay**、**手動繳費**（轉帳後上傳截圖）
- 繳費前顯示費用明細與聲明，含手續費說明（教會吸收 or 繳費者承擔，可設定）
- 付款完成後系統自動更新繳費狀態，無需人工確認
- 後台**收款對帳報表**：篩選活動 / 日期 / 狀態，四張彙整卡 + 明細表 + 匯出 CSV

**權限設計：**
超管可設定「財務管理群組標籤」（例如：會計），只有該群組的同工才看得到金流設定與對帳報表，避免帳號資訊外流。

**市場比較：**

| 工具 | 限制 |
|------|------|
| LINE Pay 個人收款 | 無法整合報名系統，對帳靠截圖 |
| Google 表單 + ATM 轉帳 | 人工核帳，無自動通知 |
| Accupass 收款 | 手續費高（3-5%）、資料不在教會系統內 |
| **本系統** | **與報名整合、自動更新狀態、對帳報表一鍵匯出 CSV** |

### 模塊 5：天父日記（AI 靈修日記）⭐ 獨創功能

**解決的問題：** 靈修容易三分鐘熱度；牧者無法有效陪伴會友靈修生活。

這是本系統**最具原創性**的模塊，目前市面上沒有完全相同的產品。

**功能說明：**

每天打開，會看到：
1. **今天的聖經段落**，經文全文直接顯示
2. **段落背景**（AI 生成，1-2 句幫你進入狀況）
3. **反思引導**：AI 根據今天這幾節具體經文，提出 2-3 個問題
4. **靈修書寫區**：寫完按儲存，永久鎖定，保留最真實的感動

**月曆追蹤：**
- 🟢 綠色 = 今天有寫
- 🔴 紅色 = 有進度但沒寫（可補寫）
- ⬜ 灰色 = 還沒到的日子

**靈修星空（獨創視覺化）：**

每寫一篇靈修，天空就多一顆星。隨著篇數累積，星星逐漸連成圖案：

| 篇數 | 星空圖案 |
|------|----------|
| ~50 篇 | 十字架 |
| ~140 篇 | 聖靈鴿子 |
| ~200 篇 | 耶穌畫像輪廓 |
| ~390 篇（一整年） | 完整星圖 |

**牧養查閱：**
- 會友可主動授權牧者查閱自己的靈修記錄（完全自願）
- 牧者看到授權名單，按日期翻閱，更了解會友內心世界
- 讓牧者禱告更有方向，不是只有表面的點頭問好

**管理員後台：**
- 上傳讀經進度表，設定每天讀哪段
- 統計今天共有幾人登入靈修、進度剩幾天

**市場比較：**

| 工具 | 限制 |
|------|------|
| YouVersion / Bible App | 閱讀為主，AI 引導反思功能薄弱，無牧養查閱機制 |
| 靈修筆記本 App（如 DayOne） | 個人使用，無讀經進度整合、無牧養連結 |
| 一般 AI 聊天（如 ChatGPT） | 每次重新對話，無歷史累積，無視覺化激勵 |
| **本系統** | **AI 反思 + 歷史累積 + 星空激勵 + 牧養授權查閱，全球首創整合** |

**市場比較：**

## 四、系統整體優勢總結

### 低門檻

| 項目 | 本系統 |
|------|--------|
| 會友端 | 只需有 LINE，點一下登入 |
| 管理端 | 中文介面，不需要技術背景 |
| 部署端 | 部署精靈引導，不需工程師進駐 |

### 跨平台

| 平台 | 狀態 |
|------|------|
| iPhone | ✅ 加到主畫面即 App |
| Android | ✅ 加到主畫面即 App |
| 電腦瀏覽器 | ✅ 完整功能 |
| 平板 | ✅ 自動適應版面 |

### 整合性

本系統最大的優勢不是任何單一功能，而是**把分散的事情整合在一起**：
- 會友資料只需填一次
- 報名資料自動關聯會友名單
- 通知自動觸發，不需人工提醒
- 靈修日記與牧養關懷直接連結

### 原創功能

以下功能在市面上無完全相同的產品：

1. **天父日記 × 靈修星空**：AI 引導靈修 + 視覺化星空進度 + 牧養授權查閱，三合一整合
2. **小組週報彙整**：訪客填報（無需登入）+ 主任牧師全教會一頁總覽

### 模塊 X：首頁三層客製化 ⭐ 新功能

**解決的問題：** 青年小組和長執關心的功能不同，所有人看到一樣的首頁，重要的反而被淹沒。

**三層架構：**

```
全域預設（系統預設全顯示）
    ↓ 超管可設定
小組預設（青年 / 兒童 / 樂團各自不同）
    ↓ 個人可覆蓋
個人設定（我自己想要的樣子）
```

**可客製化的內容：**

| 層級 | 可控制 |
|------|--------|
| 區塊（6 個） | Hero 橫幅、本週待辦、近期活動、靈修空間、功能卡片、本週資訊 |
| 功能卡片 | 每張卡片個別顯示/隱藏、個人可拖曳排序 |

**市場比較：**

| 工具 | 限制 |
|------|------|
| 一般系統 | 所有人介面相同 |
| SharePoint / Teams | 企業向，非教會設計，複雜 |
| **本系統** | **超管設小組預設，個人可再覆蓋，三層 fallback 自動生效** |
