# 整合型教會行政系統 展示報告書

## 系統簡介

整合型教會行政系統是一套專為台灣教會設計的數位化管理平台，將原本四個獨立的子系統整合為單一統一的 Web 應用程式。系統以 LINE 帳號登入為核心，讓每一位會友都能透過手機輕鬆使用，不需要額外下載 App，也不需要記憶帳號密碼。

系統涵蓋教會日常行政的方方面面：從活動報名、小組管理、靈修日記，到讀經進度追蹤、禱告代禱、福音關懷、行事曆、差勤管理、檔案分享……超過 15 個功能模組，約 13,000 行程式碼，全部整合在同一個入口。

## 系統架構

### 技術選型

| 層次 | 技術 |
|------|------|
| 後端框架 | Python Flask（Blueprint 模組化架構） |
| 資料庫 | Supabase PostgreSQL（雲端託管） |
| 登入機制 | LINE Login OAuth2 |
| 檔案儲存 | Cloudflare R2（S3 相容） |
| AI 功能 | Groq（llama-3.3-70b，主）/ Gemini 2.0 Flash（備援） |
| 部署平台 | Render（自動化 CI/CD） |
| 前端 | 純 HTML + Jinja2 模板 + 原生 JavaScript（無框架） |
| PWA | 支援加入主畫面，離線圖示 |

### 架構設計原則

本系統採用 Flask Blueprint 架構，每個功能模組獨立為一個 Blueprint，可單獨啟用或停用。整合自以下四個原始子系統：

1. **event-registration-**：活動報名、簽到、會友管理（基礎核心）
2. **church-data-hub**：檔案分享（Cloudflare R2 儲存）
3. **tianfu-diary**：靈修日記 + AI 反思引導
4. **cell_reporter**：小組週報回報（由 Django 移植至 Flask）

## 使用者角色

系統定義四種角色，各有不同的功能權限：

| 角色 | 說明 | 代表功能 |
|------|------|---------|
| 一般會友 | 使用 LINE 登入的所有人 | 報名活動、靈修日記、個人行程、代禱牧田 |
| 訪客 | 尚未成為正式會友 | 限縮功能（不顯示禱讀本、部分頁面） |
| 管理員（Admin） | 教會同工 | 活動管理、行事曆、課程、週報、小組回報查閱 |
| 超級管理員（Super Admin） | 主任牧師等 | 會員管理、系統設定、資料匯出匯入、部署精靈 |
| 牧者（Pastor） | 小組長或牧者 | 查閱授權會友的靈修日記 |
| 同工（Staff） | 教會同工 | 差勤申請審核、出席統計查閱 |

## 功能模組總覽

### 一、門戶首頁（Portal）

每位用戶登入後進入個人化的功能入口。

**特色功能：**
- 以圖卡（Tile）方式展示所有可用功能，支援管理員自訂卡片名稱、順序
- 頂部顯示今日讀經進度提示
- 即時顯示本週各聚會人數統計（成人主日、兒童主日、禱告會、晨禱），自動抓取當週日期，未填時顯示 0
- 站內通知中心，顯示未讀數量
- 每日祝福經文顯示

### 二、活動報名與管理

讓會友線上報名教會活動，管理員輕鬆追蹤報名狀況。

**會友端功能：**
- 瀏覽開放中的活動，線上一鍵報名
- 報名後可查看報名狀態、繳費狀態
- 支援外部連結報名（無需登入）
- 繳費截止前自動站內提醒通知

**管理員功能：**
- 建立活動（名稱、時間、地點、費用、人數上限、開放狀態）
- 自訂報名表單欄位（文字、選擇、問卷等）
- 匯出報名名單為 Excel
- 手動發送活動通知給所有報名者
- 關閉/開放報名

### 三、電子簽到

活動現場快速簽到，告別紙本點名。

**功能：**
- 生成簽到 QR Code（可貼在門口或投影）
- 會友掃碼或自行輸入身分驗證簽到
- 管理員即時查看已簽到名單
- 支援外部連結（無需事先登入）

### 四、行事曆

整合教會行事、系統活動、課程堂次和個人行程的統一月曆視圖。

**四種來源：**
1. **教會行事**：管理員手動新增（支援 CRUD），可廣播通知全體會友
2. **系統活動**：從活動管理模組自動顯示
3. **課程堂次**：從門訓課程模組自動顯示
4. **個人行程**：登入用戶私人行程（CRUD，只有本人可見）

**通知整合：**
- 每次進入首頁自動觸發懶式提醒檢查
- 個人行程和教會行事均支援「提前 N 天提醒」
- 已讀過的提醒，下次日期到時可重新觸發

### 五、門訓課程管理

系統化管理教會的門徒訓練課程，追蹤出席與認證。

**功能：**
- 建立課程（包含多個堂次、排程時間）
- 會友線上報名課程
- 管理員登記各堂次出席紀錄
- 自動計算出席率、認證進度
- 課程認證管理（查看、補登認證記錄）
- 教材庫存追蹤（進銷存）
- 小組門訓出席總覽
- 路由約 50 個，是系統最大的功能模組

### 六、天父日記（靈修日記）

結合 AI 技術的每日靈修記錄系統。

**會友端功能：**
- 每日讀經進度顯示（連接和合本聖經全文 JSON）
- AI 即時生成段落背景說明（一到兩句，快取共用）
- AI 根據當日具體經文生成 2-3 個反思引導問題
- 靈修書寫區（寫完一鍵鎖定，不可修改，可複製分享）
- 月曆視圖追蹤每日靈修狀態（已寫/未寫/未到/選中）
- 書卷瀏覽：依聖經書卷查看靈修記錄，可展開閱讀
- 靈修星空：Canvas 動畫，每篇靈修點亮一顆星，漸進形成十字架 → 聖靈鴿子 → 耶穌畫像（約 390 篇完成）
- 牧養查閱授權：自願授權指定牧者/小組長查閱自己的日記

**管理員後台：**
- 上傳/手動填寫/下載讀經進度表（xlsx 格式）
- 讀經進度狀況統計（今日進度、涵蓋範圍、總天數、距到期天數）
- 牧者白名單管理（設定/移除可查閱日記的人員）

**AI 技術：**
- 優先使用 Groq（llama-3.3-70b-versatile），備援 Gemini 2.0 Flash
- 反思問題和段落背景各自有記憶體快取，同一段經文全教會共用，不重複呼叫

### 七、小組週報回報

小組長每週回報聚會狀況，牧者線上查閱。

**小組長功能：**
- 填寫週報（聚會日期、出席人數、活動記錄）
- 記錄組員出席狀況（小組聚會、主日崇拜、讀禱歸狀態）

**公開回報（無需登入）：**
- 成人主日聚會人數
- 兒童主日聚會人數
- 禱告會人數
- 晨禱人數

**管理員功能：**
- 管理小組、組員清單
- 查閱各週回報統計
- 週報數據自動顯示於首頁統計卡

### 八、代禱牧田

會友彼此代禱的平台。

**功能：**
- 發表代禱需求（設定公開/私密）
- 為他人禱告（按禱告按鈕）
- 禱告次數顯示
- 留言回應
- 管理員可設定代禱分類

### 九、福音探索

協助福音關懷與跟進管理。

**功能：**
- 自訂福音探索問答卡（Q&A 內容）
- 記錄福音詢問來源與跟進狀況
- 管理員查閱詢問記錄

### 十、禱讀本訂購系統

小組式禱讀本團購管理流程。

**完整流程：**
1. 管理員建立訂購批次（經卷名稱、作者、定價、截止日）
2. 各小組透過系統登記購買數量
3. 管理員 key-in 或各小組自行填報
4. 系統自動統計各小組數量與合計金額
5. 管理員列印簽收條（自動觸發瀏覽器列印）
6. 現場簽收後在系統標記已簽收

**特色：**
- 每個小組每批次唯一登記（Upsert 邏輯防止重複）
- 所有修改都寫入 LOG 表，可追溯
- 封面圖片上傳（Supabase Storage）
- 管理員 key-in 介面支援 ±1 數量按鈕

### 十一、每週週報

PDF 週報管理與展示。

**功能：**
- 管理員上傳每週週報 PDF
- 會友線上查看最新週報
- 歷史週報存檔

### 十二、檔案分享（資料交換中心）

整合自 church-data-hub 的雲端檔案管理。

**功能：**
- 多層資料夾結構
- 上傳/下載各類檔案
- 儲存空間監控（Cloudflare R2）
- 管理員可設定儲存空間上限

### 十三、會員管理

**功能（超管限定）：**
- 查看所有會員資料
- 設定會員角色（管理員、牧者、同工、超管）
- 手動建立帳號：為未使用 LINE 登入的會友手動建立帳號
- 整合 LINE：手動帳號與 LINE 帳號合併（移轉所有歷史紀錄後刪除舊帳號）
- 搜尋會員（姓名、LINE 暱稱）

### 十四、差勤管理

同工請假與加班管理。

**功能：**
- 同工申請請假/加班
- 管理員審核（核准/拒絕）
- 請假記錄查詢

### 十五、留名單

新朋友留名單照片管理。

**功能：**
- 上傳訪客留名單照片
- 管理員查閱

### 十六、站內通知系統

全系統統一的通知機制。

**通知類型：**
| 類型 | 觸發時機 |
|------|---------|
| 繳費提醒 | 活動/課程費用截止前 N 天，進首頁時自動觸發 |
| 行程提醒 | 個人行程/教會行事前 N 天，進首頁時觸發 |
| 活動通知 | 管理員手動發送給報名者 |
| 課程通知 | 管理員手動發送給選課者 |
| 教會行事廣播 | 建立教會行事時勾選，立即廣播給全體會友 |
| 系統公告 | 超管廣播給全體用戶 |
| 更新日誌 | 發布系統更新通知 |

**去重機制：** 只跳過「未讀」的同類通知；用戶讀過後，下次日期到時可重新提醒。

## 技術亮點

### 1. 懶式通知觸發
通知提醒不依賴背景排程，而是在用戶進入首頁時自動觸發檢查（`/api/notifications/check-reminders`）。優點是零伺服器排程成本，缺點是用戶需進入首頁才能收到提醒，適合教會這種「定期打開」的使用模式。

### 2. AI 快取架構
天父日記的 AI 功能（段落背景、反思問題）以書卷+章節範圍為 Key 做記憶體快取。全教會所有使用同一段經文的人共用同一份 AI 結果，既節省 API 費用，又確保快速回應。

### 3. 讀經進度雙層讀取
進度表優先從 Supabase 資料庫讀取（5 分鐘 TTL），資料庫為空時 Fallback 到 `plan.xlsx` 本地檔案。確保在資料庫尚未設定時也能正常運作。

### 4. 靈修星空動畫
純 Canvas 實作，無任何第三方圖表庫。每篇靈修記錄點亮一顆星，累積到 50 篇形成「十字架」，140 篇形成「聖靈鴿子」，200 篇形成「耶穌畫像」（三個階段共約 390 顆星）。超過 390 篇的靈修記錄散落為背景星空。

### 5. 功能模組化開關
透過環境變數（`ENABLE_COURSES`、`ENABLE_CALENDAR`、`ENABLE_BULLETIN` 等）可以個別啟用或停用每個功能模組，方便不同規模的教會按需使用。

### 6. LINE 登入整合
所有用戶資料以 LINE User ID 為主鍵識別，登入流程完全自動化，會友無需記帳號密碼。超管可為尚未使用 LINE 的會友手動建立帳號，待對方使用 LINE 登入後再合併（移轉所有歷史資料）。

## 適用對象

本系統適合：
- **中型台灣教會**（約 50-500 位會友）
- 使用 LINE 作為主要溝通工具的教會
- 希望數位化管理但不想維護複雜 IT 基礎設施的教會
- 有技術同工可協助部署的教會

## 部署與維運

- **平台**：Render（PaaS，自動從 GitHub 部署）
- **資料庫**：Supabase（免費方案可支援中小型教會）
- **檔案儲存**：Cloudflare R2（S3 相容，費用極低）
- **AI 費用**：Groq 提供免費額度，一般教會使用量不會超額
- **維護成本**：月費約 0-20 USD（視用量），無需專職 IT 人員
