首次登入與管理員設定
操作指引 ▼
1
系統部署完成後，使用 LINE 帳號登入。首次登入的帳號將自動建立 users 資料。
2
至個人資料頁複製自己的 LINE User ID（格式：U 開頭的 32 位字串）。
3
方案 A（推薦）：將此 User ID 填入 ADMIN_LINE_USER_IDS 環境變數，重新部署後即為管理員。
方案 B：在 Supabase SQL Editor 執行上方 SQL，直接設定資料庫欄位。
4
取得管理員權限後，至 後台管理 → 小組管理 建立小組與組員， 並在 聚會設定（/cell-report/admin/meeting-settings）設定各聚會的星期幾與堂數。 若有額外聚會類型（如青年崇拜），可點「＋ 新增聚會」自訂，系統會自動在 Dashboard 與回報頁面反映。
5
（可選）至 後台管理 → 門戶設定 調整首頁顯示的功能卡片。

📁 Cloudflare R2 檔案分享
設定 R2 後，教會可上傳檔案與資料夾共享。未設定則檔案功能停用（其餘功能不受影響）。
設定步驟：
1. 前往 Cloudflare Dashboard → R2 → 建立 Bucket
2. 在「Manage R2 API Tokens」建立 Token（Object Read & Write 權限）
3. 將 Endpoint / Access Key / Secret Key / Bucket Name 填入對應環境變數
🤖 AI 功能（天父日記 AI 反思引導）
設定 Groq API Key 後，天父日記可啟用 AI 反思引導功能。Groq 提供免費額度，足夠小型教會使用。
取得方式：前往 console.groq.com 免費申請 API Key，填入 GROQ_API_KEY。

Render 部署設定
部署參數 ▼
Build & Start 設定
Build Command pip install -r requirements.txt
Start Command gunicorn "app:create_app()" --limit-request-line 8190
Runtime Python 3 (auto-detected from runtime.txt)
Plan Free tier 可用（冷啟動約 30 秒）；付費 Starter 無冷啟動問題
重要環境變數
Render 免費方案預設不設定 RENDER 旗標， 需手動在 Environment 新增 RENDER=true 讓系統知道自己在 production 環境 （啟用 HTTPS-only session cookie）。
RENDER=true FLASK_SECRET_KEY=（用 python 生成的 32 位 hex 字串） LINE_CHANNEL_ID=（LINE Developers 取得） LINE_CHANNEL_SECRET=（LINE Developers 取得） LINE_REDIRECT_URI=https://你的網域.onrender.com/auth/callback SUPABASE_URL=https://你的專案.supabase.co SUPABASE_KEY=（Supabase anon key） CHURCH_NAME=你的教會名稱 CHURCH_SHORT_NAME=教會簡稱 ADMIN_LINE_USER_IDS=U你的LINE_USER_ID
自訂網域
1. 在 Render Service → Settings → Custom Domains 新增你的網域
2. 在 DNS 設定 CNAME 指向 Render 提供的值
3. 等待 SSL 憑證自動簽發（約 10 分鐘）
4. 更新 LINE_REDIRECT_URI 為新網域並重新部署

新增一間教會的完整 SOP
1
請教會端申請 Supabase
前往 supabase.com → 建立帳號 → 新增 Project（免費方案即可）
完成後到 Project Settings → API，複製：
 • Project URL → 給你填 SUPABASE_URL
 • anon public key → 給你填 SUPABASE_KEY
2
請教會端申請 LINE Login Channel
前往 developers.line.biz → 用教會負責人的 LINE 帳號登入
建立 Provider → 新增 LINE Login Channel
完成後複製：
 • Channel ID → 給你填 LINE_CHANNEL_ID
 • Channel Secret → 給你填 LINE_CHANNEL_SECRET
⚠️ Callback URL 先留空，等你建完 Render 服務後再回來填
3
在你的 Render 帳號建新 Web Service
Dashboard → New + → Web Service → 選你的 GitHub repo
Name 取好識別名稱（例：church-grace-system）
Region 選 Singapore，Branch 選 main
4
填入環境變數（Environment Variables 區塊）
# ── 必填 ────────────────────────────────── RENDER=true FLASK_SECRET_KEY=<python -c "import secrets; print(secrets.token_hex(32))"> LINE_CHANNEL_ID=<教會端提供> LINE_CHANNEL_SECRET=<教會端提供> LINE_REDIRECT_URI=https://<你命名的服務>.onrender.com/auth/callback SUPABASE_URL=<教會端提供> SUPABASE_KEY=<教會端提供> CHURCH_NAME=<教會全名，例：天恩浸信會> CHURCH_SHORT_NAME=<教會簡稱，例：天恩堂> ADMIN_LINE_USER_IDS=<教會管理員的 LINE User ID> # ── 天父日記 AI（教會端各自申請，填教會自己的 key）── GROQ_API_KEY=<教會端申請 groq.com 取得> GEMINI_API_KEY=<教會端申請 aistudio.google.com 取得，備援用>
5
建立 Render 服務並取得網址
點「Create Web Service」→ 等待 Build 完成（約 3～5 分鐘）
部署成功後取得網址：https://church-xxx.onrender.com
6
回到 LINE Developers 填 Callback URL
進入該教會的 LINE Login Channel → LINE Login 分頁
在 Callback URL 填入：https://你的服務.onrender.com/auth/callback
儲存後即可使用 LINE 登入
7
在 Supabase SQL Editor 建立資料表
進入教會的 Supabase 專案 → SQL Editor → 貼上 Step 3 的 schema.sql → 執行
回到本精靈 Step 3 確認所有資料表顯示 ✓
8
首次登入 & 設定管理員
用你的 LINE 登入系統 → 進入個人資料頁確認你的 LINE User ID
將 ID 填入 Render 的 ADMIN_LINE_USER_IDS 環境變數 → 儲存 → 重啟
重新登入後即可進入後台管理
9
