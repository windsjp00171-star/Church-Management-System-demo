# 教會行政系統 — 部署說明書（給教會窗口）

> 這是一套整合型教會行政系統，用 **LINE 登入**即可使用，會友手機就能操作。
> 本文件分兩部分：**(A) 系統有哪些功能**、**(B) 上線前請教會幫忙準備的資料**。
> 標示 ⭐ 為上線必備，其餘為可日後再開的功能。

---

# Part A — 系統功能總覽

### 1. 活動與報名 ⭐
- **活動報名**：建立教會活動，可自訂報名表單欄位（姓名、電話、餐點選擇等）、設人數上限、報名截止時間。
- **候補與白名單**：額滿可開候補；限定名單才能報名的活動可設白名單。
- **付費活動**：可串接線上付款（綠界 / LINE Pay）或手動繳費。
- **外部報名**：非會友也能用連結報名（不需 LINE 登入）。

### 2. 電子簽到 ⭐
- **QR Code 簽到**：活動現場掃碼即可簽到。
- **即時簽到看板**：同工可即時看到誰到了、到了幾人。
- **現場大螢幕**：有人簽到時播放慶祝動畫，炒熱氣氛。
- **簽到名單匯出**：一鍵匯出 Excel。

### 3. 會員管理 ⭐
- 會友 / 訪客分類、服事標籤（敬拜、招待…）、權限分層（管理員 / 超級管理員 / 牧者 / 同工）。
- 可手動建立沒有 LINE 的會友，日後再與 LINE 帳號合併。

### 4. 小組牧養
- **小組回報單**：小組長每週線上回報出席、組員四面向（屬靈 / 家庭 / 工作 / 健康）狀況，牧者可彙整查閱。
- **聚會人數統計**：主日（成人 / 兒童）、禱告會、晨禱、自訂聚會的出席人數歷史紀錄。

### 5. 門訓 / 課程
- 課程與堂次管理、學員報名、出席追蹤、結業認證、教材庫存進銷存。

### 6. 靈命造就
- **天父日記**：個人靈修日記，搭配 **AI 反思引導**，附讀經進度表。
- **代禱麥田**：張貼代禱事項、彼此回應與代禱（可匿名）。
- **每日經文**：每日祝福經文卡。

### 7. 對外關懷與佈道
- **福音探索**：福音 Q&A 卡片 + 慕道朋友線上留言，同工後台跟進。
- **留名單**：新朋友留名單照片管理。
- **禱讀本訂購**：開訂購、各小組登記本數、列印簽收條、QR 登記連結。

### 8. 行政與溝通
- **教會行事曆**：活動、課程、教會行事一頁總覽。
- **每週週報**：上傳與發布週報 PDF。
- **公告廣播**：一鍵發站內通知給全體會友。
- **差勤系統**：全職同工請假 / 加班線上申請與審核、特休補休時數。

### 9. 系統與資料安全
- **資料備份中心**：管理員可隨時把所有核心資料匯出成 Excel(CSV) / ZIP / 離線 HTML，**即使系統停擺也拿得回資料**。
- **檔案分享**：教會內部檔案上傳分享（選配，需雲端儲存）。
- **首頁自訂**：功能卡片、快捷連結、各小組首頁可自行調整。

# Part B — 上線前請教會幫忙準備的資料

> 下列「擁有權」都歸**教會**，我（維護方）只負責程式與部署、**不持有也不接觸會友資料**。
> 帳號建議用**教會自己的信箱**申請，方便日後長期掌管。

## B-1. 必備（沒有就無法上線）⭐

- [ ] **教會名稱**：正式全名 + 簡稱（例：恩典浸信會 / 恩典）
- [ ] **Supabase 帳號**（資料庫）
  - 用教會信箱註冊 https://supabase.com，現場我協助開「正式專案」並建好資料表
- [ ] **LINE Developers 帳號**（會友登入用）
  - 現場開一個 **LINE Login** channel，取得 Channel ID / Channel Secret
- [ ] **Render 帳號**（主機 / 網站）
  - 用教會信箱註冊 https://render.com，現場我協助部署
- [ ] **管理員名單**
  - 誰是管理員（可多位）；其中**指定一位「超級管理員」**＝教會主責行政（系統最高權限交給教會）
  - 管理員本人需在現場用 LINE 登入一次（用來取得身分識別碼）

## B-2. 選配（看教會要不要開這些功能）

- [ ] **AI 天父日記引導** → 需 **Groq 帳號/金鑰**（免費，https://console.groq.com）
- [ ] **檔案分享功能** → 需 **Cloudflare R2** 儲存空間（帳號 + bucket）
- [ ] **線上付款** → 綠界 / LINE Pay 商店帳號（可日後再接）
- [ ] **隱私權政策連結**（若教會已有，可顯示在登入頁）
- [ ] **教會 Logo / 主視覺**（美化首頁用，可日後補）

## B-3. 上線後要陸續輸入的「內容資料」（可先用 Excel 備好，非當天必須）

- [ ] **小組清單**：各小組名稱、小組長、組員名單
- [ ] **服事團隊 / 部門**分類（敬拜、招待、兒主…）
- [ ] **各場聚會時間**：主日場次、禱告會、晨禱等
- [ ] **第一波活動**：想先開放報名的活動資訊
- [ ] **門訓課程**清單（若要用課程模組）
- [ ] **福音探索問題** / **禱讀本品項**（若要用對應模組）

# Part C — 平台帳號、流量額度與收費說明（申辦前先看）

> 明天請窗口**當場用教會自己的信箱申辦**以下帳號，擁有權即歸教會。
> 重點：**起步全部可走免費方案**；只有「用量變大」或「想更穩定」時才需要付費。

### 各平台一覽

| 平台 | 用途 | 免費額度（起步夠用） | 何時才需付費 |
|------|------|----------------------|--------------|
| **LINE Login** | 會友登入 | 完全免費、無人數上限 | 不需付費 |
| **Supabase** | 資料庫（存所有資料） | 500MB 資料庫、1GB 檔案、每月 5 萬活躍用戶 | 資料量極大或要每日自動備份才升級（約 US$25/月） |
| **Render** | 主機 / 網站 | 每月 750 小時（夠 1 個服務 24 小時運作） | 想「不休眠、零等待、更穩」時升級 Starter（約 US$7/月） |
| **Cloudflare R2**（選配） | 檔案儲存 | 每月 10GB 儲存 + 大量免費流量 | 檔案存很多才會超量 |
| **Groq**（選配） | AI 靈修引導 | 免費額度足夠一般教會使用 | 幾乎用不到付費 |

### 「流量 / 額度」白話解釋（給窗口聽得懂）

- **Supabase 的額度**看的是「**存了多少資料**」和「**多少人登入使用**」。一般中小型教會的會友、活動、回報資料，離 500MB 還很遠，**多年都用不完免費額度**。
- **Render 的額度**看的是「**主機開機時數**」。免費方案每月 750 小時，剛好夠一個網站整月不關；所以**一間教會一個系統 → 免費就夠**。唯一缺點是閒置 15 分鐘會休眠，下次開啟等 30–60 秒（已用 UptimeRobot 保活改善）。
- **R2 / Groq** 是選配功能才會用到，沒開就完全不產生費用。

### 收費原則（請當面講清楚，最重要）

1. **我（系統維護方）不收任何系統開發、部署、維護費用。**
2. 上述第三方平台費用**由教會自行負擔**，但**起步階段幾乎都在免費額度內**，多數教會長期零成本。
3. 任何「要不要升級付費」的決定權都在教會，升級不影響資料與設定，隨時可升可降。

# Part C-2 — LIFF 是什麼？要不要申辦？

**LIFF＝讓會友在「LINE App 裡面」開啟系統時，自動登入、不用再按登入鍵的功能。**

- **不設定 LIFF**：系統照常運作。會友開啟時點一下「用 LINE 登入」按鈕即可（在任何瀏覽器都通）。
- **有設定 LIFF**：當會友從 LINE 聊天室點連結、在 LINE 內建瀏覽器開啟時，會**自動登入**，體驗更順（長輩少一個步驟）。

**結論**：LIFF 是**錦上添花的選配**，不是必備。建議**第一天先不設、確認登入正常後**，日後想優化長輩體驗時再到 LINE Developers 開一個 LIFF app、把 `LINE_LIFF_ID` 填進 Render 即可，不需改任何程式。
