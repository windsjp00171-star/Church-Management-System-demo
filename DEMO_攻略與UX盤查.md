# 教會行政系統 — Demo 攻略 + UX 盤查

> 用途：Demo 前備戰小抄。每個模組含「準備什麼資料 → 怎麼走最亮眼 → 最可能被問倒的問題＋體面答法」。
> 最後附「Demo 前最好先補的地雷」與「全站 UX 待改善」。

---

## 〇、Demo 前的總準備（共通）

**必備測試帳號（4 種視角）**
- 一般會友、訪客（member_type=visitor）、管理員/超管、牧者（is_pastor）＋ 至少一個小組長。

**最容易在現場出糗的三個地雷（建議 demo 前先確認）**
1. **週報、Office 檔、福音卡靠 Google Docs Viewer 預覽** — 若現場網路擋 Google 或斷線，PDF/PPT 會開天窗。**備案：先截圖或備一份可下載連結。**
2. **AI 反思引導要等 2–5 秒（逾時 10 秒）** — 現場網路差會卡。**備案：先在同網路試一次讓快取建立，或準備一張截圖。**
3. **DEMO_MODE 底部洽詢橫幅在手機上會佔約 50–60px** — 手機展示時記得它會遮一點底部內容。

**現場若 LINE 登入失敗**：預留一個已登入的瀏覽器分頁，不要現場才登入。

## 一、會員端核心

### 1. 登入 + Onboarding
- **準備**：一個全新 LINE 帳號（展示首登 Onboarding 填真實姓名流程）、一個已登入帳號。
- **怎麼走**：在 LINE 內開 → LIFF 幾乎零摩擦自動登入 → 首登導到 Onboarding 填名 → 進首頁。
- **可能被問倒**：
  - 「能用帳號密碼登入嗎？」→ **不行，只支援 LINE 登入**（這是刻意設計，教會場景 LINE 普及率最高，省去記密碼）。
  - 「換手機要重新登入嗎？」→ 用同一 LINE 帳號即可，資料都在。

### 2. 首頁 Portal
- **準備**：3–4 個開放報名活動（含海報、不同狀態）、1 個本週小組討論（YouTube）、1 份最新週報。
- **亮點**：Hero 動畫 + 活動卡片；**個人化首頁**（卡片可隱藏、拖曳重排，三層：管理員全域→小組預設→個人）。
- **可能被問倒**：
  - 「不同人看到的首頁一樣嗎？」→ 不一樣，依角色/小組顯示不同卡片，個人還能自訂。
  - 「我報名的活動怎麼不見了？」→ 活動日期過了就從「我的活動」收起，只顯示即將到來的。

### 3. 活動報名 ⭐ 招牌
- **準備三種活動最有戲**：
  - A 快速報名（無自訂欄位、一鍵報名）
  - B 標準報名（含**條件邏輯欄位**：選某項才跳出下一題 + 餐點選擇）
  - C **額滿活動 + 開啟候補**（先塞 2–3 筆候補）
- **亮點**：條件邏輯動態欄位、候補機制、一鍵快速報名。
- **可能被問倒**（重要）：
  - 「候補的人，前面有人取消會自動遞補嗎？」→ **會！系統自動把候補第一位升為正式報名**（已驗證）。
  - 「活動改時間，已報名的人會自動收到通知嗎？」→ **目前不會自動通知，要管理員手動發**（誠實講，這是目前待補的點）。
  - 「能匯出報名名單嗎？」→ **能，後台一鍵匯出 Excel**（含自訂欄位、中文表頭）。
  - 「外部的人（非會友）能報名嗎？」→ 能，若活動開啟外部報名，可走免登入表單。

### 4. 電子簽到
- **準備**：一個開放簽到的活動 + QR Code（用第二支手機顯示或列印）；幫測試帳號先報名 2 筆展示「多筆選擇」。
- **亮點**：掃碼即簽、多筆報名清楚列出選擇、簽到成功 confetti 動畫。
- **可能被問倒**：
  - 「QR Code 給別人掃，會不會被代簽？」→ **目前會**（掃碼即可簽，無額外身分驗證）。可坦白說「適合信任環境，若要防代簽是下一步可加的」。
  - 「簽到錯了能取消嗎？」→ 目前要管理員後台處理，前台無自助取消。

### 5. 個人資料 / 牧養小組申請
- **準備**：2–3 個牧養小組、2 位牧者。
- **亮點**：申請加入小組 → 管理員核准流程；天父日記授權給指定牧者（隱私可控）。
- **可能被問倒**：
  - 「能同時加入多個牧養小組嗎？」→ 目前設計一對一（一個牧養小組）。服事團隊標籤（group_tags）才能多選。

### 6. 站內通知
- **亮點**：報名成功即時通知；繳費提醒（進首頁自動檢查）。
- **可能被問倒**：
  - 「通知會推到 LINE 嗎？」→ **目前只有站內通知，沒有 LINE 推播**（誠實講，常被問，可說是 roadmap）。
  - 「能關掉某類通知嗎？」→ 目前無偏好設定。

### 7. 行事曆 / 8. 週報
- **行事曆準備**：教會行事 + 活動 + 課程堂次 + 個人行程各幾筆，展示顏色分類。
- **週報準備**：3–4 份週報，最新一份設為當週。
- **可能被問倒**：「週報能下載嗎？」→ 內嵌預覽為主，下載非預設；「行事曆能匯出 Google Calendar 嗎？」→ 目前不行。

## 二、靈修與牧養

### 9. 天父日記 ⭐⭐ 最招牌（AI 功能）
- **準備（關鍵）**：
  - `diary_plan` 要有**今天**的讀經進度（如 2026-06-10 詩篇 23:1-6）。
  - 確認 GROQ_API_KEY 或 GEMINI_API_KEY 有設定且有額度。
  - 一個會友帳號 + 一個牧者帳號。
  - **demo 前先在現場網路點一次 AI，把快取建立起來**。
- **怎麼走**：會友登入 → 今日讀經自動展開逐節經文 → 點「✦ 反思引導」→ AI 生成 2–3 個反思問題 → 寫日記 → 儲存並鎖定 → 日曆標記「已寫」→ 切牧者帳號「查閱成員靈修」看到該會友日記。
- **可能被問倒**（隱私類，最常問）：
  - 「我的日記別人看得到嗎？」→ **預設只有自己**，除非你主動授權給特定牧者。
  - 「牧者能看到全部人的日記嗎？」→ **不行，只能看主動授權給他的人**。
  - 「AI 會把我的日記拿去訓練嗎？」→ 反思問題是**根據經文生成、不送出你的日記內容**；用的是 Groq（不訓練用戶資料）。可補一句「教會可自選 AI 供應商」。
  - 「日記寫錯能改嗎？」→ **儲存後會鎖定不能改**（設計上鼓勵真誠記錄），目前要改需管理員處理。

### 10. 代禱麥田
- **準備**：預建 5–10 則代禱，做出不同回應數（展示🌰種子→🌱→🌿→🌾成熟的麥穗成長）。
- **亮點**：麥穗隨回應數成長、蒙應允亮起、匿名選項。
- **可能被問倒**：
  - 「匿名是真的匿名嗎？」→ 對其他會友顯示「匿名」，**但系統後台仍有紀錄**（管理用），不是完全匿名，要誠實講。
  - 「有人回應我的代禱會通知我嗎？」→ 目前需自己重整查看，無推播。

### 11. 福音探索
- **準備**：5–10 張信仰問答卡、預設表單欄位、3–5 筆舊詢問記錄。
- **亮點**：**免登入**公開頁、3D 翻卡、訪客填詢問 → 自動通知超管 → 指派小組長跟進。
- **可能被問倒**：「訪客送出後能查進度嗎？」→ 目前無確認信/追蹤碼；「能自動分配跟進的人嗎？」→ 需人工指派。

### 12. 門訓學程
- **準備**：1 門課 5 堂（缺勤上限設 1）、1 個課程類別、學員預先報名，其中 1 人已簽到 4 次（準備現場簽第 5 次觸發自動完訓）。
- **亮點**：簽到後**自動判斷完訓並頒發認證**；認證體系支援「回鍋」重修。
- **可能被問倒**：
  - 「課程能限報名人數嗎？」→ **目前沒有人數上限欄位**，需管理員手動關閉報名。
  - 「認證會過期嗎？」→ 不會，永久持有。
  - 「能下載結業證書嗎？」→ 目前只在「我的學程」顯示徽章，無 PDF 證書。

### 13. 禱讀本訂購
- **準備**：1 筆未截止訂購（價格、截止日、封面圖）、3–4 個小組的登記（部分標記已交付）。
- **亮點**：小組化登記、修改履歷溯源、列印簽收清單、交付追蹤。
- **可能被問倒**：
  - 「同一小組多人登記會怎樣？」→ **以小組為單位，後登記的會覆蓋前一筆**（一個小組一筆）。要提醒小組由一人負責登記。
  - 「能分批交付嗎？」→ 目前只有「已交付/未交付」兩種狀態。

## 三、管理後台

### 14. 活動管理
- **亮點**：海報拖曳上傳預覽、報名名單一鍵 Excel 匯出（自訂欄位、簽到/繳費狀態都帶）。
- **可能被問倒**：
  - 「報表能匯出嗎？」→ 報名名單 **Excel（.xlsx）可以**，CSV 目前金流對帳頁才有。
  - 「活動有人報名後還能刪嗎？」→ 能，但會連帶刪掉所有報名紀錄，**無二次確認、無還原**（demo 時別示範刪除）。

### 15. 小組回報
- **準備**：2–3 個小組 + 各幾位組員、指派小組長/牧者帳號。
- **亮點**：三步驟填報引導、牧者可看跨組統計、各類聚會（主日/兒童/禱告會）開關。
- **可能被問倒**：
  - 「能補填上週嗎？改已送出的嗎？」→ **都可以**（用日期選擇進舊週報重新編輯）。
  - 「小組長看得到別組嗎？」→ **看不到**，權限嚴格限制只能看自己帶的組。
  - 「能自動每週提醒小組長交報告嗎？」→ **目前沒有排程提醒**（誠實講，需人工或未來加排程）。

### 16. 同工差勤
- **亮點**：依到職日自動算特休、請假/加班/補休餘額一頁看清、管理員審核。
- **可能被問倒**：「有分層簽核嗎？」→ 沒有，任一管理員都可核准；「特休會自動結轉嗎？」→ 目前無結轉清零邏輯，需留意。

### 17. 金流對帳
- **亮點**：支援綠界/LINE Pay/手動繳費、對帳報表含手續費試算、可匯出 CSV。
- **可能被問倒**：
  - 「支援退款嗎？」→ **系統無退款按鈕**，需人工撥款後手動改狀態。
  - 「能自動催繳嗎？」→ 有繳費提醒，但**無排程自動發送**，需手動。
  - 「支援哪些付款？」→ 綠界、LINE Pay、手動轉帳，僅台幣。

### 18. 資料匯出/匯入
- **亮點**：整個教會資料打包成 ZIP（11 個模組可選）、可用於遷移或備份。
- **可能被問倒**：「備份有加密嗎？」→ **無，ZIP 是明文含 LINE ID 等個資**，要妥善保管（重要，誠實講）。「支援增量備份嗎？」→ 目前都是全量。

### 19. 其他管理（會員管理/留名單/設定精靈/更新日誌）
- **會員管理**：搜尋、封鎖、升降管理員。被問倒點：無批量匯入會員、封鎖無自動通知。
- **留名單**：拍照上傳新朋友名單。被問倒點：一次一張、無分類。
- **設定精靈**：⚠️ **目前任何登入同工都進得去，會看到環境變數設定**——這是**安全建議要修的點**（限超管）。
- **更新日誌**：存在 `changelogs.json`，多教會共用同一份（無法各教會分開）。

## 四、檔案分享 + 全站

### 20. 檔案分享
- **準備**：幾個不同格式檔案（圖片/PDF/音訊/影片）、一個密碼保護檔、管理員帳號看儲存空間儀表板。
- **亮點**：三種上傳（按鈕/拍照/拖曳）、角色×使用者×分組權限、密碼保護、分享連結（1h/24h/7d 失效）、音訊直接播放。
- **可能被問倒**：
  - 「空間多大？」→ 預設 10GB，可在 settings 調（目前無後台 UI，要改設定值）。
  - 「檔案有防毒掃描嗎？」→ **沒有**（誠實講）。
  - 「刪除能還原嗎？」→ **不行，永久刪除，無回收站**。
  - 「刪資料夾裡面的檔案會一起刪嗎？」→ **不會，檔案會變成沒有資料夾的孤兒檔**（已知待改善）。

### 全站體驗（做得好的）
- 所有模組統一 base.html、配色字體一致，**沒有「換了個網站」的違和感**。
- 404/500/403 都有設計過的友善頁面，無死路。
- 右下角「?」說明書、首登脈衝提示、訪客未完成資料的提醒橫幅。
- 手機/LINE WebView 相容處理完整（拍照上傳用 `<label for>` 繞過 LIFF 限制）。

## 五、Demo 前「最好先補」的小地雷（投報率高）

| 優先 | 項目 | 為什麼 demo 前修 | 工作量 |
|------|------|------------------|--------|
| 高 | 設定精靈限超管才能進 | 同工能看到金鑰設定頁，被內行人看到很尷尬 | 小 |
| 高 | AI 端點加 rate limit | Demo 公開站，AI 額度可能被路人刷光 | 小 |
| 中 | 取消報名/刪活動加二次確認 | 現場誤點刪掉資料無法復原 | 小 |
| 中 | 空狀態加「目前沒有資料」提示 | 沒資料的區塊會空白，看起來像壞掉 | 小 |
| 低 | 部分 presigned URL 補 expires | 安全細節 | 小 |

## 六、全站 UX 三大待改善（中長期，非 demo 必修）

1. **通知只有站內、無 LINE 推播也無分類** — 使用者漏看率高，是最常被問也最有感的缺口。
2. **多步驟操作/刪除缺一致的二次確認與還原** — 報名、代禱、檔案、活動刪除多為硬刪除無回收站。
3. **缺全文搜尋與報表圖表** — 代禱/日記/會員無搜尋；牧者統計、差勤、金流只有數字無圖表。

## 七、Demo 建議流程（約 12 分鐘）

1. **開場（2 分）**：訪客 → LINE 登入 → Onboarding → 首頁，展示「零摩擦入門」。
2. **會友日常（4 分）**：活動報名（條件欄位+候補）→ 簽到 confetti → **天父日記 AI 反思引導**（招牌）。
3. **牧養協作（3 分）**：牧者查閱授權日記 → 小組回報三步驟 → 代禱麥田麥穗成長。
4. **管理效率（2 分）**：報名名單一鍵匯出 Excel → 金流對帳一覽。
5. **收尾（1 分）**：強調多教會獨立部署、資料各自獨立、開發者不碰教會資料（隱私賣點）。

**萬用誠實答題公式**：做不到的功能 →「目前這部分是手動/還沒做，已經在規劃中」，不要硬凹。你的系統**整合度高、隱私設計清楚**，這兩點是最強賣點，被問倒時可以拉回這裡。
